# Marketing y Publicaciones — Curso: Cómo usar Claude
### Estrategia de contenido y banco de prompts para promocionar el curso completo de Claude

---

## Contexto del curso (para usar en todos los prompts)

```
Nombre: Cómo usar Claude — De cero a avanzado
Módulos: Chat / Proyectos / Claude Code / Agentes / Conectores y MCP
Nivel: Básico a avanzado (un solo curso progresivo)
Audiencia: Profesionales que ya oyeron hablar de Claude o lo usan
           ocasionalmente pero no aprovechan sus funciones avanzadas
Propuesta de valor: Dominar el ecosistema Claude (claude.ai + Claude Code)
                    para resolver tareas reales y construir agentes por rol
Diferenciador: No es un curso genérico de IA — es específico del ecosistema
               Claude, con Proyectos, Artifacts, Claude Code (CLI/IDE),
               agentes con roles definidos (C-Level → Analista) y MCP
Herramientas: claude.ai (Free / Pro / Team) y Claude Code (CLI/IDE)
```

---

# Estrategia de contenido

## Los 5 pilares editoriales

| Pilar | Qué comunica | Formatos ideales |
|---|---|---|
| **Demo en vivo** | Claude haciendo algo impresionante en pantalla | Reel, YouTube, TikTok |
| **Comparación** | Claude vs ChatGPT vs Gemini — cuándo usar cuál | Carrusel, tabla, hilo |
| **Caso de uso real por rol** | Cómo un C-Level, Director, Manager o Analista usa Claude | Post narrativo, video, email |
| **Tutorial paso a paso** | Enseña a usar una función específica del ecosistema | YouTube, carrusel, stories |
| **Agente del día** | Un agente real configurable en <30 min | Post + GIF, hilo, video corto |

## Regla de proporción

```
3 publicaciones de demo o tutorial (valor puro)
: 1 publicación de conversión al curso
```

La audiencia para este curso ya usa herramientas digitales. No necesita convencerse de que la IA existe — necesita ver que el ecosistema Claude resuelve problemas que ChatGPT no resuelve igual de bien (Proyectos compartidos, Claude Code en su repo, agentes con scope claro).

## Prompt 1 — Post educativo: la diferencia entre Claude y los demás

> **Para:** LinkedIn, Instagram (carrusel o texto)
> **Pilar:** Comparación

```
Actúa como experto en herramientas de productividad con IA, con experiencia
probando ChatGPT, Gemini y Claude en contextos de trabajo real.

Escribe un post para LinkedIn que explique de forma honesta y específica
cuándo Claude es mejor opción que ChatGPT o Gemini, y cuándo no lo es.

Estructura:
- Gancho: una afirmación que rompa el lugar común de "todas las IAs son iguales"
- 3 situaciones concretas donde Claude tiene ventaja real:
  (documentos largos, Proyectos con contexto persistente compartido,
  Claude Code en repos reales con permissions y hooks)
- 1 situación donde ChatGPT o Gemini ganan a Claude
  (honestidad = credibilidad)
- Cierre: invitación a aprender a sacarle partido específicamente al
  ecosistema Claude completo (no solo el chat)

Tono: análisis honesto de alguien que usa las tres herramientas,
no publicidad de una sobre otra.
Largo: máximo 220 palabras.
No uses el lenguaje de "la mejor IA del mercado" — sé específico y medible.
```

## Prompt 2 — Carrusel: el ecosistema Claude explicado visualmente

> **Para:** Instagram, LinkedIn (carrusel de 8 diapositivas)
> **Pilar:** Tutorial paso a paso

```
Actúa como diseñador de contenido educativo especializado en tecnología
para audiencias no técnicas.

Crea el guion de un carrusel de 8 diapositivas que explique el ecosistema
Claude completo: Chat, Proyectos, Claude Code, Agentes y Conectores/MCP.

Estructura:
- Diapositiva 1 (portada): "Claude no es solo un chat. Es un ecosistema
  de 5 capas. La mayoría solo conoce 1."
- Diapositiva 2: Chat — qué es, para qué, 2 casos de uso cotidiano
- Diapositiva 3: Proyectos — contexto persistente entre conversaciones,
  1 ejemplo de proyecto real (gestión de clientes / blog / RRHH)
- Diapositiva 4: Claude Code — qué es, dónde vive (terminal/IDE), para
  quién (devs y data analysts), 1 ejemplo sorprendente (refactor con plan mode)
- Diapositiva 5: Agentes — el espectro completo (Personalizar → Proyectos
  → Skills → Subagents). Pequeña matriz de quién hace qué por rol
- Diapositiva 6: Conectores y MCP — Claude leyendo Gmail, Drive, Figma,
  tu base de datos. 1 ejemplo concreto (briefing matutino automatizado)
- Diapositiva 7: Cuándo usar cada capa (tabla visual: situación → capa ideal)
- Diapositiva 8 (cierre): llamada a la acción suave hacia el curso

Para cada diapositiva:
- Título (máximo 8 palabras)
- Texto principal (máximo 20 palabras)
- Nota para el diseñador: qué ícono, ilustración o captura de pantalla usar
```

## Prompt 3 — Reel / TikTok: demo en pantalla de una función de Claude

> **Para:** Instagram Reels, TikTok, YouTube Shorts (45–60 segundos)
> **Pilar:** Demo en vivo

```
Actúa como guionista de contenido educativo para video corto,
especializado en demos de herramientas digitales.

Escribe el guion de un reel de 50 segundos que muestre en pantalla
cómo usar los Proyectos de Claude para no perder el contexto
de trabajo entre conversaciones.

Escenario: alguien que trabaja con el mismo cliente cada semana y
siempre tiene que repetirle a Claude quién es ese cliente y qué quiere.

Estructura:
- 0–5 seg: gancho ("¿Le repites lo mismo a Claude cada vez que abres un chat?")
- 5–15 seg: el problema — muestra cómo se ve tener que repetir el contexto
- 15–40 seg: la solución — cómo crear un Proyecto con instrucciones
  persistentes en 3 pasos (muestra la pantalla paso a paso)
- 40–50 seg: resultado + dato de impacto + CTA ("Link en bio para aprender
  el ecosistema Claude completo: Chat, Proyectos, Claude Code y agentes")

Formato: narración en voz / texto en pantalla / acción en la app (3 columnas).
Tono: rápido, sin tecnicismos, que un profesional no técnico lo entienda.
Incluye 2 opciones de gancho alternativo para hacer A/B test.
Incluye 3 opciones de thumbnail con texto de máximo 4 palabras.
```

## Prompt 4 — Hilo de LinkedIn: casos de uso reales por profesión y nivel

> **Para:** LinkedIn (hilo de 7 partes)
> **Pilar:** Caso de uso real por rol

```
Actúa como creador de contenido B2B en LinkedIn especializado en
productividad con IA para profesionales no técnicos.

Escribe un hilo de LinkedIn de 7 posts sobre cómo distintos profesionales
usan Claude en su trabajo real, uno por publicación. Incluye el nivel de
seniority en cada caso (C-Level / Director / Manager / Analista) para
mostrar que el ecosistema sirve a todos.

Perfiles a cubrir (uno por post):
1. CEO de una scaleup: usa Proyectos con conector a Gmail/Calendar para
   un Daily Executive Briefing automatizado
2. Director de Marketing: define un Proyecto compartido del equipo con
   guías de marca y casos pasados; mide ahorro de horas/semana
3. Manager de Producto: usa un Proyecto con docs de OKRs para triage
   de feedback de clientes
4. Analista de Datos: configura un subagent SQL en Claude Code conectado
   a Snowflake vía MCP — no sabía hacer queries complejos antes
5. Abogado in-house: usa Chat con documentos largos para extraer
   cláusulas de riesgo en contratos
6. Engineering Manager: usa Claude Code con hooks para que tests y linter
   corran siempre antes de cada commit
7. Asistente ejecutiva: usa Proyectos para gestionar la agenda y
   comunicaciones de su director con tono consistente

Para cada post:
- Etiqueta del rol y seniority al inicio
- Gancho: situación específica de esa profesión (que los demás se identifiquen)
- El problema que tenía con las IAs genéricas
- Cómo lo resolvió con Claude (qué capa del ecosistema, cómo lo configuró)
- Resultado concreto: tiempo ahorrado, calidad mejorada, frustración evitada

Post final (8): resumen + invitación al curso.

Tono: concreto y creíble. Que parezca un caso real, no un testimonio fabricado.
Largo por post: máximo 150 palabras.
```

## Prompt 5 — Video largo para YouTube: tutorial del módulo Proyectos

> **Para:** YouTube (10–15 minutos)
> **Pilar:** Tutorial paso a paso

```
Actúa como guionista de tutoriales de YouTube para herramientas de
productividad con IA, dirigido a profesionales no técnicos.

Escribe el guion completo de un video de YouTube de 12 minutos titulado:
"Cómo usar los Proyectos de Claude para nunca perder el contexto de tu trabajo"

Audiencia: profesionales que ya usan Claude en Chat pero no conocen los Proyectos.

Estructura:
- Hook (0–30 seg): pregunta que identifica el dolor
  ("¿Cuánto tiempo pierdes repitiendo el contexto de tu trabajo
  a Claude al inicio de cada conversación?")
- Qué vas a aprender hoy (30–60 seg): los 3 superpoderes de los Proyectos
- Tutorial parte 1 (1–4 min): crear un Proyecto y escribir las instrucciones
  (demo en pantalla paso a paso)
- Tutorial parte 2 (4–7 min): subir documentos al Proyecto y cómo Claude
  los usa como contexto
- Tutorial parte 3 (7–10 min): organizar múltiples conversaciones dentro
  del Proyecto sin que se mezclen
- Caso real (10–12 min): muestra un ejemplo completo de flujo de trabajo
  profesional de principio a fin (puede incluir un Proyecto compartido en Team)
- CTA final (12 min): resumen + link al curso completo (que también cubre
  Claude Code y agentes por rol)

Formato del guion: narración / acción en pantalla / tiempo estimado (3 columnas).
Incluye: título SEO + 5 variantes de título + descripción del video (150 palabras)
+ 5 tags de YouTube relevantes.
```

## Prompt 6 — Email de lanzamiento del curso

> **Para:** Lista de correo
> **Pilar:** Conversión directa

```
Actúa como copywriter especializado en emails de lanzamiento de cursos
de herramientas digitales para profesionales.

Escribe el email de anuncio del curso "Cómo usar Claude — de cero a avanzado"
para una lista de profesionales que ya usan alguna herramienta de IA
(ChatGPT, Gemini u otra) pero no son usuarios activos del ecosistema Claude.

Estructura:
- 3 opciones de asunto (una con pregunta, una con número, una con beneficio directo)
- Cuerpo del email:
  - Párrafo 1: el problema — usan IA pero sienten que no están aprovechando
    todo su potencial (identificación sin culpa)

    casos reales)
  - Cierre: CTA + una razón para actuar ahora que no sea urgencia falsa

Tono: como alguien que encontró algo útil y quiere compartirlo,
no como un vendedor de cursos.
Largo: máximo 280 palabras en el cuerpo.
Español latinoamericano neutro.
```

## Prompt 7 — Secuencia de stories: tutorial visual de una función

> **Para:** Instagram Stories / WhatsApp Status
> **Pilar:** Demo en vivo

```
Actúa como estratega de contenido para Instagram Stories educativos
sobre herramientas digitales.

Crea una secuencia de 6 stories que enseñen paso a paso cómo usar
la ejecución de código y los Artifacts de Claude para analizar un

Contexto: la audiencia son profesionales que trabajan con datos pero
no tienen habilidades técnicas. La secuencia debe sorprenderlos

Estructura:
- Story 1: Gancho — "¿Sabías que puedes analizar cualquier hoja de datos
  con Claude sin tocar Excel ni saber programar?"

- Story 4: Paso 2 — el prompt exacto que escribes para pedir el análisis
  con visualizaciones

- Story 6: CTA — "Esto es solo una de las 5 capas del ecosistema Claude.
  El curso te enseña Chat, Proyectos, Claude Code y agentes. Link en bio."

Para cada story:
- Texto principal (máximo 12 palabras)
- Sugerencia visual (fondo, captura de pantalla, sticker o ícono)
- Si hay narración en video: qué decir (máximo 10 segundos de habla)
```

```
Actúa como estratega de marketing de contenidos para el lanzamiento
de un curso sobre el ecosistema Claude.

"Cómo usar Claude — de cero a avanzado".

  no solo un chat (Chat + Proyectos)

  (C-Level, Director, Manager, Analista)

- Tema central de la semana
- 4 publicaciones (martes a viernes):
  plataforma + formato + idea del contenido en 1 oración + CTA

Formato de salida: tabla con columnas:
Semana / Día / Plataforma / Formato / Contenido / CTA

Nota: el lunes es día de planificación interna. No publicar.
Adapta el tono de cada publicación según la plataforma:
LinkedIn más formal, Instagram más visual, WhatsApp más conversacional.
```

# Tips para usar estos prompts

## Bloque de contexto — agrega esto al inicio de cada prompt

```
Contexto del instructor:
- Nombre: [tu nombre]
- Experiencia con Claude: [cuánto tiempo llevas usándolo, en qué contextos]
- ¿Has usado Claude Code? [sí / no — ajusta los prompts del Módulo 3]
- Plataforma de venta del curso: [nombre o "por definir"]
- Precio: [precio o "por definir"]
- Formato: [online / presencial / asincrónico / en vivo]
- Duración total: [horas o módulos]
- Fecha de lanzamiento: [fecha]
- Redes donde tienes más audiencia: [LinkedIn / Instagram / etc.]
- Tono de tu marca personal: [profesional / cercano / técnico / divulgativo]
- Audiencia objetivo principal: [no técnicos / técnicos / mixta]
```

## Flujo de ejecución recomendado

```
SEMANA -2 (preparación)
→ Prompt 12: calendario editorial completo
→ Prompt 11: descripción del curso para la landing
→ Prompt 6: email de lanzamiento

SEMANA -1 (calentamiento)
→ Prompt 1: post comparativo Claude vs otros (genera debate)
→ Prompt 8: primer mito vs realidad

SEMANA DE LANZAMIENTO
→ Prompt 2: carrusel del ecosistema Claude completo (5 pilares)
→ Prompt 3: reel de demo en pantalla
→ Prompt 7: secuencia de stories
→ Prompt 9: mensajes para comunidades

POST-LANZAMIENTO
→ Prompt 4: hilo de LinkedIn por rol y profesión
→ Prompt 5: video largo de YouTube (evergreen)
→ Prompt 10: repurposing del mejor contenido generado
```

## Métricas por tipo de contenido

| Contenido | Métrica principal | Señal de que funciona |
|---|---|---|
| Demo en pantalla (reel) | Guardados + reproducciones | Guardados > 2% del alcance |
| Carrusel educativo | Guardados + compartidos | Guardados > 3% del alcance |
| Hilo LinkedIn | Comentarios de otras profesiones/roles | "Yo también lo usaría para..." |
| YouTube tutorial | Retención promedio | Retención > 50% en los primeros 3 min |
| Email lanzamiento | Tasa de apertura + clics al curso | Apertura > 30%, clics > 5% |
| Stories con CTA | Deslizamientos al link | CTR > 5% |
| Post por rol | Comentarios "soy [rol], esto me sirve" | Validación del segmento |
