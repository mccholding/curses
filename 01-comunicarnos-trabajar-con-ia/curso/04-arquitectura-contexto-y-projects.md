# Módulo 4 · Arquitectura del contexto y Projects

**Duración:** 60 minutos.

## Objetivos

- Decidir cuándo crear un Project o espacio persistente.
- Organizar fuentes e instrucciones para un mismo tema.
- Separar conversaciones por objetivo.
- Diseñar continuidad sin arrastrar historiales innecesarios.

## Cuándo crear un Project

Tiene sentido cuando el trabajo:

- dura varios días o semanas;
- reutiliza las mismas fuentes;
- requiere una voz, proceso o formato estable;
- incluye varias conversaciones relacionadas;
- será compartido por un equipo.

No hace falta para una traducción puntual, una pregunta aislada o una tarea que no volverá a repetirse.

## Anatomía recomendada

```text
Nombre
Propósito
Usuarios
Fuentes disponibles
Reglas permanentes
Formatos de salida
Decisiones tomadas
Acciones prohibidas
Criterios de calidad
Fecha de revisión
```

## Plantilla de instrucciones

```markdown
# Propósito
Este espacio ayuda a [audiencia] a producir [entregables].

# Contexto estable
[organización, producto, proceso o situación]

# Fuentes
Usa primero las fuentes oficiales. Distingue hechos, inferencias y recomendaciones.

# Forma de trabajar
1. Confirma el objetivo y entregable.
2. Señala información faltante.
3. Produce un borrador.
4. Verifica contra los criterios.
5. Pide aprobación antes de acciones externas.

# Reglas
- No mezclar información entre clientes.
- No inventar cifras, políticas o citas.
- Identificar versiones y fechas.

# Formato
[idioma, tono, estructura, longitud]
```

## Una conversación por objetivo

```text
Project: Lanzamiento Q4
├── Investigación
├── Estrategia
├── Propuesta comercial
├── Contenido
└── Seguimiento y decisiones
```

El Project comparte lo estable. Cada conversación mantiene un objetivo y un historial más limpio.

## Estrategia de continuidad

Cuando una conversación ya cumplió su función o acumuló demasiado historial:

1. pedir un resumen de decisiones, fuentes y pendientes;
2. guardar el resumen si será reutilizado;
3. abrir una conversación nueva con un objetivo específico;
4. referenciar únicamente el resumen y fuentes necesarias;
5. archivar o nombrar claramente la conversación anterior.

## Auditoría periódica

```markdown
## Mantener
[instrucciones, reglas y fuentes vigentes]

## Actualizar
[datos o decisiones con fecha]

## Archivar
[duplicados, versiones anteriores, ruido]

## Separar
[objetivos que necesitan conversaciones independientes]

## Criterio de suficiencia
La IA tiene contexto suficiente cuando puede producir [resultado]
sin repetir [información estable] ni consultar material irrelevante.
```

## Práctica guiada

Crear un espacio real con:

- instrucciones;
- entre tres y siete fuentes;
- una conversación de análisis;
- una conversación de producción;
- un resumen de continuidad;
- una prueba sobre información inexistente.

## Entregable

Project o diseño equivalente con inventario de fuentes, estructura de conversaciones, reglas de actualización y auditoría de contexto.
