# Mapa del curso · Cómo usar Claude: de cero a avanzado

## La progresión

```text
BLOQUE 1 · USAR
Claude, modelos, Chat, archivos, imágenes y análisis
        ↓
BLOQUE 2 · PRODUCIR
Artifacts, documentos, visualizaciones y prototipos
        ↓
BLOQUE 3 · ORGANIZAR
Proyectos, bases de conocimiento e instrucciones persistentes
        ↓
BLOQUE 4 · CONECTAR
Conectores, MCP y acceso controlado a herramientas
        ↓
BLOQUE 5 · DELEGAR
Tareas de varios pasos, agentes, trabajo en escritorio y programación
        ↓
BLOQUE 6 · GOBERNAR
Privacidad, permisos, revisión humana, equipos y medición de valor
        ↓
ESPECIALIZACIÓN · CONSTRUIR
Claude Code, Skills, subagentes, hooks, plugins y CI/CD
```

## Ruta A · Claude para profesionales

| Módulo | Tema | Duración | Producto del alumno |
|---:|---|---:|---|
| 0 | Diagnóstico y mapa del ecosistema | 1 h | Mapa personal de casos de uso |
| 1 | Chat, modelos, archivos y análisis | 2 h | Flujo de análisis verificable |
| 2 | Artifacts y creación de entregables | 2 h | Artifact funcional o documento reutilizable |
| 3 | Proyectos y bases de conocimiento | 2 h | Proyecto configurado con instrucciones y fuentes |
| 4 | Investigación, datos y fuentes | 2 h | Informe con evidencia y criterios de verificación |
| 5 | Conectores y MCP para profesionales | 2 h | Flujo conectado de bajo riesgo |
| 6 | Delegación, agentes y tareas repetitivas | 2 h | Delegación con criterio de terminado y checkpoint humano |
| 7 | Equipos, seguridad, adopción y proyecto final | 3 h | Sistema de trabajo documentado y demo |

**Duración total sugerida:** 16 horas.

## Ruta B · Especialización Claude Code

| Bloque | Módulos | Enfoque |
|---|---|---|
| Fundamentos operativos | 1–2 | Instalación, loop agéntico, contexto, permisos y flujo diario |
| Personalización | 3–4 | Settings, comandos, hooks iniciales y Skills |
| Conexión y orquestación | 5–7 | MCP, subagentes, agent teams, worktrees y plugins |
| Automatización | 8–10 | Headless, CI/CD, Agent SDK, hooks empresariales y tareas programadas |
| Flujo profesional | 11–12 | Git/GitLab, troubleshooting y runbook interno |

**Duración total sugerida:** 24–36 horas más proyecto integrador.

## Formatos posibles

### Curso abierto para profesionales

- Dictar únicamente la Ruta A.
- Evitar terminal y código.
- Usar casos de operaciones, marketing, educación, consultoría y liderazgo.

### Curso para equipos técnicos

- Ruta A comprimida en 4 horas como nivelación.
- Especialización Claude Code completa.
- Proyecto final sobre un repositorio real del equipo.

### Programa in-company mixto

- Todos completan módulos 0–5 de la Ruta A.
- Perfiles no técnicos continúan con delegación y gobernanza.
- Perfiles técnicos pasan a Claude Code.
- Ambos grupos presentan un único proceso empresarial desde perspectivas complementarias.

## Criterios de salida

Una persona termina la **Ruta profesional** cuando puede:

- Elegir la superficie correcta de Claude para cada tarea.
- Trabajar con archivos y fuentes sin confiar ciegamente.
- Crear un Proyecto con contexto mantenible.
- Diseñar un flujo conectado con permisos mínimos.
- Delegar una tarea con entregable, verificación y aprobación humana.

Una persona termina la **Especialización Claude Code** cuando puede:

- Operar Claude Code con seguridad en un repositorio real.
- Configurar CLAUDE.md, reglas, permisos y Skills.
- Integrar herramientas mediante MCP.
- Dividir trabajo con subagentes y equipos.
- Automatizar tareas con hooks, modo headless y CI/CD.
- Distribuir la configuración mediante plugins y documentación.
