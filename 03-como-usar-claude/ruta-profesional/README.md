# Ruta profesional · Claude aplicado

## Propósito

Aplicar los fundamentos del Curso 02 para producir entregables, organizar conocimiento, conectar herramientas y delegar trabajo profesional dentro de Claude.

## Prerrequisito

Completar `02 · Fundamentos de trabajar con IA` o demostrar dominio equivalente.

No se vuelven a impartir como teoría principal: contexto, RAG, MCP, agentes, loops, subagentes, tareas programadas y control humano.

## Audiencia

- operaciones y administración;
- consultoría;
- educación;
- marketing y contenido;
- análisis y liderazgo;
- emprendimiento y PyME.

No requiere terminal ni programación.

## Duración

- **8 horas**.
- Cuatro bloques de dos horas.
- Mínimo **70 % de práctica**.
- Un proceso real durante toda la ruta.

## Cuatro bloques

| Bloque | Enfoque | Evidencia |
|---:|---|---|
| 1 | Artifacts y entregables | Artifact funcional con criterios de aceptación |
| 2 | Projects y conocimiento | Project configurado con fuentes y continuidad |
| 3 | Conectores y permisos | flujo de bajo riesgo con matriz de acceso |
| 4 | Cowork, revisión y proyecto final | encargo completo, evidencia y manual operativo |

## Uso de los archivos existentes

1. [Ecosistema, Chat y archivos](./01-ecosistema-chat-y-archivos.md): utilizar únicamente la demostración y el diagnóstico necesarios para nivelar la interfaz.
2. [Artifacts, Projects y conocimiento](./02-artifacts-proyectos-y-conocimiento.md): núcleo de los bloques 1 y 2.
3. [Conectores, delegación y automatización](./03-conectores-delegacion-y-automatizacion.md): usar conectores como práctica; la teoría de MCP, agentes y loops se considera aprendida.
4. [Equipos, gobernanza y proyecto final](./04-equipos-gobernanza-y-proyecto-final.md): cierre, adopción, seguridad y continuidad.

## Proyecto transversal

La tarea elegida debe:

- repetirse al menos dos veces al mes;
- usar una fuente clara;
- producir un entregable reconocible;
- tener criterios objetivos;
- incluir una decisión que conserve una persona.

```text
Artifact
→ Project
→ fuente conectada
→ encargo en Cowork
→ revisión
→ sistema documentado
```

## Evidencias mínimas

- Artifact con objetivo, audiencia y pruebas;
- Project con instrucciones y fuentes;
- inventario de conocimiento;
- conexión de bajo riesgo;
- matriz de permisos;
- delegación con criterio de terminado;
- checkpoint humano;
- prueba de fallo;
- manual de continuidad y desconexión.

## Regla del instructor

Una explicación conceptual que ya existe en el Curso 02 no puede ocupar más de diez minutos. El tiempo recuperado debe dedicarse a configuración, ejecución, revisión y mejora del proyecto del alumno.
