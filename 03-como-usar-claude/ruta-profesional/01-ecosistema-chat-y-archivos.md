# Módulo 0 · Diagnóstico y mapa del ecosistema

**Duración:** 1 hora.

## Objetivos

Al finalizar, el participante podrá:

- Explicar qué lugar ocupa Claude dentro de su caja de herramientas.
- Diferenciar Chat, Projects, Artifacts, conectores y Claude Code.
- Elegir una superficie de trabajo según el tipo de tarea.
- Identificar un proceso real que desarrollará durante el curso.

## Contenido

### 0.1 Claude no es un solo producto

Presentar el ecosistema como superficies con propósitos diferentes:

| Superficie | Mejor para | No es la primera opción cuando... |
|---|---|---|
| Chat | Preguntas, redacción, análisis puntual e iteración | Necesitas contexto estable durante semanas |
| Projects | Trabajo continuo con instrucciones y fuentes compartidas | La tarea es única y sencilla |
| Artifacts | Crear documentos, visualizaciones, prototipos y herramientas reutilizables | Solo necesitas una respuesta corta |
| Conectores | Consultar o actuar sobre herramientas externas | Puedes resolverlo sin acceso adicional |
| Trabajo agéntico | Delegar tareas de varios pasos | El objetivo o criterio de terminado todavía es ambiguo |
| Claude Code | Desarrollo, repositorios, terminal y automatización técnica | El trabajo no vive en archivos técnicos o código |

### 0.2 Diagnóstico de madurez

Cada alumno clasifica su nivel actual:

1. **Usuario ocasional:** hace preguntas sueltas.
2. **Usuario productivo:** trabaja con archivos y buenos prompts.
3. **Usuario organizado:** utiliza proyectos e instrucciones persistentes.
4. **Usuario conectado:** integra herramientas y fuentes.
5. **Usuario delegador:** entrega objetivos completos y supervisa resultados.

### 0.3 Selección del proceso transversal

El proceso elegido debe tener:

- una entrada concreta;
- una secuencia reconocible;
- un entregable final;
- una forma de verificar calidad;
- límites sobre lo que Claude no debe hacer solo.

## Práctica guiada

Crear un inventario de diez tareas de la última semana y ubicarlas en una matriz:

| Tarea | Frecuencia | Tiempo | Riesgo | Superficie de Claude recomendada |
|---|---:|---:|---|---|

## Entregable

Una página titulada **Mi mapa personal de Claude**, con tres casos de uso prioritarios y el proceso transversal seleccionado.

# Módulo 1 · Chat, archivos y análisis verificable

**Duración:** 2 horas.

## Objetivos

- Utilizar Chat para producir resultados controlables.
- Trabajar con documentos, imágenes y datos.
- Separar hechos, inferencias y recomendaciones.
- Diseñar una revisión adecuada al riesgo del resultado.

## Contenido

### 1.1 Prompting como repaso, no como curso repetido

El participante ya debe conocer prompting. Se utiliza una fórmula breve:

```text
Objetivo + contexto + fuente + entregable + restricciones + verificación
```

Ejemplo:

```text
Analiza el reporte adjunto para la directora de operaciones.
Usa únicamente el contenido del documento.
Entrega: resumen ejecutivo, tres hallazgos, dos riesgos y preguntas abiertas.
Cita la página que respalda cada hallazgo.
Si algo no aparece en la fuente, marca “no disponible”.
```

### 1.2 Trabajar con archivos

Practicar con:

- PDF y documentos extensos;
- hojas de cálculo;
- imágenes y capturas;
- varias fuentes relacionadas;
- archivos con información incompleta o contradictoria.

La pregunta pedagógica no es solo “¿qué puede leer Claude?”, sino:

> ¿Qué información necesita, cuál sobra y cuál es la fuente de verdad?

### 1.3 Estrategia de análisis en cuatro pasos

1. **Inspeccionar:** describir el archivo y sus límites.
2. **Extraer:** recuperar datos o fragmentos relevantes.
3. **Analizar:** encontrar patrones, riesgos y relaciones.
4. **Verificar:** conectar cada conclusión con evidencia.

### 1.4 Control de calidad

Usar una tabla de confianza:

| Tipo de salida | Riesgo | Revisión mínima |
|---|---|---|
| Ideas y borradores | Bajo | Revisión de utilidad y tono |
| Resúmenes internos | Medio | Comparar puntos clave con la fuente |
| Cifras, citas o decisiones | Alto | Verificar cada dato crítico |
| Contenido legal, médico o financiero | Muy alto | Validación profesional independiente |

## Práctica guiada

Analizar un documento real en tres rondas:

1. Prompt inicial.
2. Corrección del enfoque y formato.
3. Verificación contra fragmentos exactos de la fuente.

## Caso por audiencia

- **PyME:** comparar cotizaciones de proveedores.
- **Corporativo:** resumir un reporte y preparar preguntas para dirección.
- **Educación:** analizar resultados de una evaluación por competencia.
- **Marketing:** extraer mensajes y restricciones de un brandbook.
- **Consultoría:** convertir entrevistas en hallazgos y temas recurrentes.

## Entregable

Un **flujo de análisis verificable** que incluya prompt, fuente, salida, criterios de revisión y versión corregida.

## Evaluación rápida

El alumno explica:

1. Qué parte de la salida proviene directamente de la fuente.
2. Qué parte es inferencia de Claude.
3. Qué decisión sigue siendo responsabilidad humana.
