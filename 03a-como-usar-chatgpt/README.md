# 03A · Cómo usar ChatGPT: de cero a avanzado

Curso organizado como una ruta progresiva para dominar ChatGPT como plataforma de trabajo, no solamente como un chat.

**Fecha de corte del contenido:** 18 de julio de 2026.

> ChatGPT cambia con rapidez. Los nombres, límites y disponibilidad de algunas funciones dependen del plan, el país, el dispositivo y la configuración del espacio de trabajo. Antes de cada cohorte se debe revisar la documentación oficial incluida en `fuentes-oficiales.md`.

## Lugar en la ruta

Este curso es una ruta de plataforma paralela a Claude:

1. [01 · Cómo comunicarnos y trabajar con la IA](../01-comunicarnos-trabajar-con-ia/README.md)
2. [02 · Fundamentos de trabajar con IA](../02-fundamentos-trabajar-con-ia/README.md)
3. Elegir una o ambas plataformas:
   - **03A · Cómo usar ChatGPT**
   - [03B · Cómo usar Claude](../03-como-usar-claude/README.md)

El curso 01 enseña Prompt, Context y Loop Engineering. El curso 02 explica modelos, RAG, herramientas, MCP y agentes. Este curso aplica esos principios dentro del ecosistema ChatGPT.

## Arquitectura del producto

```text
CHAT
Conversación, búsqueda, voz, imágenes, archivos y ayuda rápida.
        ↓
PROJECTS
Contexto, instrucciones, archivos y conversaciones de largo plazo.
        ↓

Experiencias especializadas, conocimiento y conexiones con datos o acciones.
        ↓
CHATGPT WORK
Investigación extensa, planificación, acciones y entregables terminados.
        ↓
TAREAS PROGRAMADAS
Trabajo recurrente, monitoreo y seguimiento proactivo.
        ↓
CODEX
Desarrollo de software, repositorios, terminal, pruebas y automatización técnica.
```

## Dos rutas

### Ruta A · ChatGPT para profesionales

Para operaciones, administración, educación, marketing, consultoría, liderazgo, análisis, ventas, creación de contenido y emprendimiento.

- **Duración sugerida:** 18 horas.
- **Formato:** 10 módulos progresivos.
- **Programación:** no requerida.
- **Resultado:** diseñar un sistema de trabajo completo con Chat, Projects, investigación, Canvas, Work, GPTs, plugins, apps y tareas programadas.

[Ver Ruta profesional](./ruta-profesional/README.md)

### Ruta B · Especialización Codex

Para desarrolladores, automatizadores y perfiles técnicos.

- **Duración sugerida:** 10 módulos de 2–3 horas más proyecto.
- **Prerrequisito:** completar la Ruta A o demostrar dominio equivalente.
- **Resultado:** usar Codex con repositorios, archivos locales, terminal, pruebas, GitHub, plugins, MCP y automatizaciones.

[Ver Especialización Codex](./especializacion-codex/README.md)

## Resultados de aprendizaje

Al finalizar la Ruta profesional, la persona podrá:

- elegir entre Chat, búsqueda, investigación profunda, Work y Codex;
- trabajar con texto, voz, imágenes, archivos y análisis de datos;
- crear Projects con instrucciones, fuentes y conversaciones organizadas;
- administrar memoria, personalización y Chats temporales;
- producir investigaciones verificables con fuentes y fecha de corte;
- crear y revisar documentos, hojas de cálculo, presentaciones y Sites;
- diseñar GPTs especializados y probarlos con casos de aceptación;
- conectar plugins y apps aplicando permisos mínimos;
- programar tareas recurrentes y monitoreo;
- aplicar controles de privacidad, aprobación y gobernanza;
- construir un proyecto profesional integrado.

## Principio pedagógico

Cada módulo incluye:

1. decisión de uso;
2. demostración;
3. práctica guiada;
4. caso profesional;
5. entregable verificable;
6. prueba de errores o límites;
7. reflexión sobre privacidad y aprobación humana.

## Relación con Claude

Los cursos no deben enseñarse como una competencia de marcas.

| Capacidad | ChatGPT | Claude |
|---|---|---|
| Contexto de largo plazo | Projects y memoria | Projects y conocimiento |
| Entregables | Canvas y ChatGPT Work | Artifacts |
| Especialización sin código | GPTs | Projects, Skills y configuraciones |
| Datos y acciones | Plugins y apps | Connectors y MCP |
| Trabajo recurrente | Tareas programadas y Work | Tareas y automatización |
| Desarrollo | Codex | Claude Code |

El objetivo es que el alumno pueda elegir la plataforma adecuada según el proceso, los datos, los controles y el resultado esperado.
