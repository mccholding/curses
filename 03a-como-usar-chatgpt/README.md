# 03A · ChatGPT aplicado

## De comprender el mapa a construir un sistema de trabajo en ChatGPT

Este curso no vuelve a enseñar los fundamentos generales de inteligencia artificial. Parte del dominio de `02 · Fundamentos de trabajar con IA` y se concentra en aplicar ese mapa dentro de ChatGPT.

**Fecha de corte del contenido:** 18 de julio de 2026.

> Los nombres, límites y disponibilidad de funciones cambian según plan, país, dispositivo y espacio de trabajo. Antes de cada cohorte se debe revisar `fuentes-oficiales.md`.

## Lugar en la ruta

```text
02 · Fundamentos de trabajar con IA
              ↓
03A · ChatGPT aplicado
              ↓
Especialización Codex, Curso 05 o Curso 07
```

El alumno no necesita completar Claude y Gemini antes o después. Las rutas de plataforma son paralelas.

## Qué no se repite

Este curso asume que la persona ya comprende:

- LLM, tokens, contexto y modelos;
- alucinaciones y verificación;
- búsqueda, RAG y fuentes;
- MCP, Skills, plugins y API;
- agentes, subagentes, loops y tareas programadas;
- niveles de autonomía y control humano.

Estos conceptos pueden recordarse brevemente para una práctica, pero no ocupan módulos teóricos.

## Ruta profesional

Para operaciones, administración, educación, marketing, consultoría, liderazgo, análisis, ventas, creación de contenido y emprendimiento.

- **Duración sugerida:** 12 horas.
- **Formato:** seis bloques aplicados.
- **Programación:** no requerida.
- **Práctica:** mínimo 70 % del tiempo.
- **Resultado:** un sistema profesional con Project, investigación, entregable, GPT, conexión y tarea recurrente.

### Seis bloques diferenciales

1. **Ecosistema y flujo multimodal:** elegir correctamente entre Chat, búsqueda, Work y archivos.
2. **Projects, memoria y continuidad:** organizar instrucciones, fuentes, conversaciones y privacidad.
3. **Deep Research:** producir investigación verificable con fuentes, contradicciones y fecha de corte.
4. **Canvas y ChatGPT Work:** convertir investigación en documentos, análisis y entregables editables.
5. **GPTs, plugins y apps:** estandarizar un método y conectarlo con información o acciones autorizadas.
6. **Tareas, gobernanza y proyecto final:** programar trabajo recurrente y documentar permisos, fallos y continuidad.

[Ver Ruta profesional](./ruta-profesional/README.md)

## Especialización Codex

Para desarrolladores, analistas técnicos y automatizadores.

- **Nivelación opcional:** entorno, permisos y contexto durable.
- **Núcleo:** comprensión de repositorios, planificación de cambios, implementación, pruebas, Git, revisión, automatización y seguridad.
- **Resultado:** cambio de software revisable con evidencia.

El Curso 02 ya introduce terminal, contexto, agentes y automatización. Codex debe profundizar en ingeniería de software, no repetir esa introducción.

[Ver Especialización Codex](./especializacion-codex/README.md)

## Proyecto transversal

Cada participante construye sobre un proceso real:

```text
Project configurado
→ investigación con evidencia
→ entregable editable
→ GPT especializado
→ app o plugin autorizado
→ tarea recurrente
→ manual de operación y continuidad
```

Evidencias mínimas:

- propósito y audiencia;
- entre tres y siete fuentes;
- instrucciones persistentes;
- separación de conversaciones;
- informe con trazabilidad;
- entregable editable;
- GPT con diez pruebas;
- matriz de permisos;
- tarea programada cuando esté disponible;
- protocolo de desconexión y recuperación.

## Regla pedagógica

Cada bloque comienza con una decisión o un proceso real, no con una definición. Debe incluir demostración, práctica, evidencia, prueba de fallo, revisión de permisos y transferencia al trabajo del alumno.
