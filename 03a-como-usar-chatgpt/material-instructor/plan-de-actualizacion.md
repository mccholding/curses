# Plan de actualización del curso

## Por qué existe

ChatGPT es un producto que cambia rápidamente. Un curso centrado en una captura de pantalla queda obsoleto. Este plan separa conceptos estables de detalles volátiles y define una revisión antes de cada cohorte.

## Fecha de corte inicial

18 de julio de 2026.

## Conceptos estables

- elegir la herramienta mínima necesaria;
- separar instrucciones, fuentes, conversaciones y memoria;
- diseñar contexto relevante;
- verificar evidencia;
- controlar permisos;
- pedir aprobación antes de acciones importantes;
- documentar tareas recurrentes;
- probar casos normales y adversariales;
- conservar responsabilidad humana.

## Elementos volátiles

Revisar siempre:

- planes y precios;
- límites de uso;
- nombres de modelos;
- disponibilidad por país;
- disponibilidad por dispositivo;
- ChatGPT Work y sus capacidades;
- Tasks y frecuencia;
- memoria;
- Canvas;
- GPTs;
- plugins, apps y sincronización;
- Codex;
- opciones de voz e imágenes;
- controles de administrador;
- políticas de privacidad y retención.

## Checklist antes de cada cohorte

### Producto

- [ ] ¿Chat, Work y Codex mantienen los mismos nombres y alcances?
- [ ] ¿Work está disponible para la audiencia?
- [ ] ¿Canvas está disponible en el dispositivo usado?
- [ ] ¿Projects tiene cambios de memoria o colaboración?
- [ ] ¿Investigación profunda mantiene las mismas fuentes y controles?
- [ ] ¿GPTs permite las capacidades utilizadas en las prácticas?
- [ ] ¿El directorio sigue usando plugins y apps?
- [ ] ¿Tasks mantiene límites, planes y frecuencia?
- [ ] ¿Codex mantiene acceso y superficies?

### Seguridad

- [ ] ¿Cambió el tratamiento de datos de consumidores?
- [ ] ¿Cambió la retención de Chats temporales?
- [ ] ¿Cambió el funcionamiento de memoria?
- [ ] ¿Las apps solicitan permisos diferentes?
- [ ] ¿Existen nuevas acciones que exijan aprobación?
- [ ] ¿Las políticas institucionales del cliente fueron incorporadas?

### Material

- [ ] Actualizar capturas.
- [ ] Comprobar enlaces.
- [ ] Ejecutar todas las prácticas.
- [ ] Revisar plantillas.
- [ ] Confirmar que el proyecto final puede realizarse con el plan disponible.
- [ ] Preparar una alternativa cuando una función no esté disponible.
- [ ] Actualizar la fecha de corte.

## Etiquetas de disponibilidad

El instructor debe marcar cada demostración:

- **Amplia:** disponible para la mayoría de los usuarios.
- **Por plan:** requiere una suscripción determinada.
- **Por espacio de trabajo:** depende del administrador.
- **Por dispositivo:** web, móvil o escritorio.
- **En despliegue:** puede no estar visible todavía.
- **Alternativa:** práctica equivalente cuando no está disponible.

## Alternativas pedagógicas

| Capacidad no disponible | Alternativa |
|---|---|
| ChatGPT Work | Chat + investigación profunda + Canvas |
| Plugins o apps | archivos simulados y diseño de permisos |
| GPT Builder | ficha de GPT y prueba manual con instrucciones |
| Tareas programadas | especificación y simulación de dos ejecuciones |
| Codex | repositorio local de práctica con otro asistente técnico |
| Canvas | documento versionado fuera de ChatGPT |

## Regla sobre modelos

No construir el curso alrededor de un nombre de modelo. Enseñar a elegir según:

- complejidad;
- velocidad;
- costo o límite;
- herramientas;
- longitud del contexto;
- precisión requerida;
- disponibilidad.

## Uso de AI_updates

La carpeta `AI_updates/` debe identificar cambios de ChatGPT que puedan afectar este curso. Sin embargo, ningún brief diario debe modificar automáticamente el programa principal. El responsable revisa la fuente oficial, prueba la función y registra el cambio antes de actualizar el contenido.
