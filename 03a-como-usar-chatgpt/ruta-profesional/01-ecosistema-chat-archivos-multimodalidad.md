# Módulo 0 · Elegir entre Chat, Work y Codex

**Duración:** 75 minutos.

## Objetivos

- Entender que ChatGPT contiene experiencias distintas.
- Elegir la superficie mínima necesaria para cada tarea.
- Evitar usar un agente de largo recorrido para una pregunta sencilla.
- Reconocer qué capacidades dependen del plan, dispositivo o espacio de trabajo.

## Las tres experiencias

| Experiencia | Úsala para | No es la primera opción cuando |
|---|---|---|
| Chat | preguntas, búsqueda, ideas, explicación y ayuda rápida | necesitas producir y revisar un trabajo extenso |
| ChatGPT Work | investigación larga, análisis, acciones y entregables terminados | basta una respuesta corta o una conversación exploratoria |
| Codex | repositorios, terminal, código, pruebas y trabajo técnico | el proceso no requiere desarrollo de software |

## Árbol de decisión

```text
¿Necesitas escribir o ejecutar software?
├── Sí → Codex
└── No
    ¿La tarea exige varios pasos, fuentes, acciones o un entregable terminado?
    ├── Sí → ChatGPT Work
    └── No → Chat
```

## Regla pedagógica

La herramienta más potente no siempre es la mejor. Elegir una experiencia excesiva puede aumentar tiempo, costo, riesgo y complejidad.

## Práctica

Clasificar veinte tareas reales entre:

- Chat;
- búsqueda;
- investigación profunda;
- Work;
- Codex;
- tarea programada;
- proceso que no debe delegarse.

## Entregable

Mapa de decisiones con cinco tareas personales o profesionales y la justificación de la superficie elegida.

# Módulo 1 · Chat, archivos y multimodalidad

**Duración:** 105 minutos.

## Objetivos

- Trabajar con texto, voz, imágenes y archivos.
- Elegir el canal que aporta contexto con menos fricción.
- Analizar información sin perder trazabilidad.
- Distinguir observación, extracción, interpretación e inferencia.

## 1.1 Chat como espacio de trabajo

Una conversación debe tener un objetivo y un entregable. Cuando aparecen objetivos distintos, conviene separar chats o mover el trabajo a un Project.

Plantilla inicial:

```markdown
# Resultado
[qué debe quedar terminado]

# Contexto
[situación, audiencia y decisión]

# Material disponible
[archivos, datos, imágenes o fuentes]

# Restricciones
[tiempo, tono, longitud, políticas]

# Criterios
[cómo se comprobará la calidad]
```

## 1.2 Archivos

Antes de pedir análisis:

1. identificar el tipo de archivo;
2. indicar qué páginas, hojas o columnas importan;
3. pedir una descripción de lo que ChatGPT detectó;
4. confirmar que la interpretación de la estructura es correcta;
5. solicitar el análisis;
6. comprobar cifras o citas importantes contra el original.

## 1.3 Imágenes

Usos profesionales:

- analizar diagramas y capturas;
- interpretar material visual;
- crear conceptos y piezas;
- editar imágenes existentes;
- producir referencias visuales;
- comparar una pieza contra criterios.

No se debe asumir que un texto pequeño, una cifra o un detalle visual fue leído correctamente sin verificación.

## 1.4 Voz y dictado

La voz es útil para:

- explicar contexto rápidamente;
- practicar conversaciones;
- pensar en voz alta;
- aprender mediante preguntas;
- capturar notas.

Proceso recomendado:

```text
Hablar sin interrupción
→ pedir un resumen del contexto
→ corregir el resumen
→ convertirlo en una instrucción estructurada
→ ejecutar la tarea
```

## 1.5 Modo de estudio

El Modo de estudio debe usarse para aprender, practicar, recibir preguntas y comprobar comprensión. No reemplaza fuentes académicas, profesores ni políticas institucionales.

## 1.6 Protocolo de análisis

```text
OBSERVAR
¿Qué contiene el material?
        ↓
EXTRAER
¿Qué información explícita se encuentra?
        ↓
INTERPRETAR
¿Qué significa dentro del contexto?
        ↓
INFERIR
¿Qué conclusión razonable puede proponerse?
        ↓
VERIFICAR
¿Qué debe confirmarse en la fuente original?
```

## Práctica guiada

Crear un informe breve a partir de:

- un archivo;
- una imagen o captura;
- una explicación por voz o texto;
- una búsqueda actualizada cuando haga falta.

El informe debe separar hechos, inferencias, incertidumbres y acciones recomendadas.

## Entregable

Flujo multimodal documentado con material de entrada, instrucción, resultado, verificación y lista de errores encontrados.
