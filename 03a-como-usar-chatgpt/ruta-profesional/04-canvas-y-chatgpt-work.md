# Módulo 4 · Canvas y revisión de entregables

**Duración:** 2 horas.

## Objetivos

- Diferenciar una respuesta de chat de un objeto editable.
- Crear documentos, código o prototipos que puedan revisarse por secciones.
- Trabajar por versiones sin perder decisiones.
- Aplicar criterios de aceptación antes de cerrar un entregable.

## 4.1 Cuándo usar Canvas

Canvas tiene sentido cuando el resultado:

- supera una respuesta breve;
- requiere edición directa;
- necesita revisión por fragmentos;
- tiene varias versiones;
- debe recibir comentarios en contexto;
- incluye código o una visualización que debe probarse.

## 4.2 Brief del entregable

```markdown
# Usuario
[quién lo utilizará]

# Problema
[qué debe resolver]

# Resultado
[documento, código, prototipo o diseño]

# Material de entrada
[fuentes, notas, plantilla]

# Estructura obligatoria
[secciones o componentes]

# Restricciones
[tono, longitud, estilo, tecnología]

# Criterios de aceptación
[condiciones verificables]

# Fuera de alcance
[qué no debe incluir]
```

## 4.3 Ciclo de edición

```text
Borrador mínimo
→ revisión estructural
→ revisión de evidencia
→ revisión de claridad
→ prueba con usuario o caso
→ correcciones
→ versión aprobada
```

No conviene pedir “mejóralo” sin indicar qué dimensión debe cambiar.

## 4.4 Tipos de revisión

- **Estructura:** orden, jerarquía y cobertura.
- **Contenido:** exactitud, fuentes y consistencia.
- **Audiencia:** lenguaje, profundidad y utilidad.
- **Cumplimiento:** restricciones y políticas.
- **Accesibilidad:** comprensión, legibilidad y formatos alternativos.
- **Funcionalidad:** ejecución, errores y comportamiento.

## Práctica guiada

Crear y revisar uno de estos entregables:

- propuesta comercial;
- informe ejecutivo;
- guía de capacitación;
- página o prototipo sencillo;
- documento operativo;
- código con pruebas básicas.

## Entregable

Canvas final más historial de decisiones: qué cambió, por qué y contra qué criterio se aprobó.

# Módulo 5 · ChatGPT Work

**Duración:** 2 horas.

## Objetivos

- Delegar tareas largas con un resultado terminado.
- Preparar contexto, plan, permisos y criterios antes de ejecutar.
- Supervisar el progreso y cambiar de rumbo.
- Revisar acciones y entregables antes de utilizarlos.

## 5.1 Cuándo usar Work

ChatGPT Work es apropiado cuando el encargo combina varios elementos:

- investigación;
- análisis de archivos o datos;
- trabajo con apps o herramientas;
- creación de documentos, hojas de cálculo, presentaciones, informes o Sites;
- ejecución prolongada;
- preguntas y aprobaciones durante el proceso;
- necesidad de un resultado listo para revisar.

Para una consulta rápida, Chat sigue siendo más eficiente.

## 5.2 Contrato de trabajo

Antes de delegar:

```markdown
# Objetivo
[resultado final]

# Contexto autorizado
[archivos, Project, apps y sitios]

# Plan esperado
[etapas o hitos]

# Entregables
[formatos y archivos]

# Criterios de revisión
[exactitud, plantilla, evidencia, estilo]

# Permisos
[qué puede leer, crear o modificar]

# Aprobaciones
[qué requiere confirmación]

# Condición de parada
[cuándo debe terminar o preguntar]
```

## 5.3 Plan antes de ejecución

El estudiante debe revisar:

- interpretación del objetivo;
- datos que utilizará;
- supuestos;
- pasos propuestos;
- acciones externas;
- entregables;
- criterios;
- riesgos.

Aprobar un plan no significa renunciar a la supervisión. Se puede intervenir, corregir o detener.

## 5.4 Producción de archivos

Work puede utilizar material de origen o plantillas para producir o editar:

- documentos;
- hojas de cálculo;
- presentaciones;
- informes;
- análisis;
- Sites.

Toda salida debe revisarse en cuatro niveles:

1. estructura;
2. evidencia y cálculos;
3. formato y editabilidad;
4. utilidad para la decisión.

## 5.5 Navegación y acciones

Cuando Work utilice un navegador o apps:

- aplicar el mínimo permiso;
- no compartir credenciales en prompts;
- confirmar acciones difíciles de revertir;
- revisar información antes de enviarla;
- detenerse ante pagos, compromisos legales o cambios críticos;
- conservar registro de fuentes y acciones.

## 5.6 Diferencia entre Canvas y Work

| Canvas | Work |
|---|---|
| edición colaborativa de un objeto | ejecución de una tarea extensa |
| el usuario trabaja directamente por secciones | el sistema planifica y avanza entre pasos |
| ideal para iterar un documento o código | ideal para reunir contexto y producir varios resultados |
| control continuo en la superficie | supervisión por plan, progreso y aprobaciones |

Pueden combinarse: Work prepara un entregable y Canvas se utiliza para la edición final.

## Práctica guiada

Convertir un conjunto de notas, archivos y datos en:

- resumen ejecutivo;
- documento editable;
- hoja de cálculo o análisis;
- presentación breve;
- registro de fuentes y decisiones.

## Prueba de control

Durante la tarea se introduce:

- una fuente contradictoria;
- un dato faltante;
- una acción que requiere aprobación;
- una plantilla con secciones obligatorias.

Work debe preguntar, detenerse o declarar la limitación en lugar de ocultarla.

## Entregable

Paquete de trabajo terminado más plan aprobado, fuentes, decisiones, controles humanos y lista de revisiones realizadas.
