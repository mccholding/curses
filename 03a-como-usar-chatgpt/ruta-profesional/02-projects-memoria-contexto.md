# Módulo 2 · Projects, memoria y arquitectura del contexto

**Duración:** 2 horas.

## Objetivos

- Crear un Project con alcance claro.
- Separar instrucciones, fuentes, conversaciones y memoria personal.
- Diseñar continuidad sin arrastrar ruido innecesario.
- Elegir entre Project, GPT, memoria y Chat temporal.

## 2.1 Cuándo crear un Project

Un Project tiene sentido cuando el trabajo:

- dura varios días o semanas;
- reutiliza fuentes;
- necesita instrucciones estables;
- contiene varias conversaciones relacionadas;
- produce versiones sucesivas;
- será compartido o retomado.

No es necesario para una pregunta aislada ni para información que no debe conservarse.

## 2.2 Anatomía recomendada

```text
Nombre
Propósito
Audiencia
Resultado esperado
Instrucciones permanentes
Fuentes de verdad
Referencias
Conversaciones por objetivo
Decisiones vigentes
Información obsoleta
Criterios de calidad
Acciones prohibidas
Reglas de actualización
```

## 2.3 Las capas del contexto

| Capa | Función | Ejemplos |
|---|---|---|
| Instrucciones del Project | reglas estables del trabajo | audiencia, tono, proceso, criterios |
| Archivos | conocimiento reutilizable | políticas, plantillas, datos, manuales |
| Conversación | objetivo específico | investigación, redacción, revisión |
| Mensaje | instrucción temporal | fecha, excepción, archivo puntual |
| Memoria | personalización entre experiencias | preferencias y contexto útil del usuario |

La memoria no reemplaza una fuente de verdad ni debe utilizarse como repositorio formal de políticas, cifras o decisiones empresariales.

## 2.4 Project frente a GPT

| Project | GPT |
|---|---|
| organiza un esfuerzo en evolución | ofrece una experiencia especializada repetible |
| reúne varias conversaciones | empieza chats nuevos con una configuración común |
| conserva archivos y contexto del proyecto | combina instrucciones, conocimiento y capacidades |
| sirve para producir trabajo | sirve para estandarizar cómo se presta una ayuda |

Un Project puede contener el trabajo de un cliente. Un GPT puede representar un método que se reutiliza con distintos clientes, siempre evitando mezclar datos.

## 2.5 Memoria

La memoria puede reducir repetición y personalizar respuestas, pero requiere control.

Buenas prácticas:

1. no guardar secretos ni datos que no deban reutilizarse;
2. revisar periódicamente lo que ChatGPT recuerda;
3. corregir información desactualizada;
4. separar preferencias personales de reglas empresariales;
5. usar Chat temporal cuando no se desea memoria ni historial persistente;
6. no asumir que eliminar un chat elimina automáticamente toda memoria relacionada.

## 2.6 Chat temporal

Úsalo cuando:

- la conversación no debe aparecer en el historial;
- no quieres que cree o utilice memorias;
- estás probando una configuración;
- manejas un caso que no debe contaminar trabajos posteriores.

No convierte automáticamente una tarea sensible en segura: siguen aplicando las políticas de datos, las integraciones y los destinatarios externos.

## 2.7 Arquitectura de fuentes

| Tipo | Tratamiento |
|---|---|
| Fuente de verdad | prioridad máxima, responsable y fecha visibles |
| Referencia | guía de estilo o ejemplo, no verdad vigente |
| Contexto | notas y opiniones que pueden contradecirse |
| Temporal | datos válidos para un periodo concreto |
| Obsoleto | archivar o retirar |

Más archivos no garantizan mejores respuestas. La meta es información mínima suficiente, vigente y recuperable.

## 2.8 Organización de conversaciones

```text
Project: Lanzamiento de producto
├── 01 Investigación
├── 02 Decisiones y supuestos
├── 03 Plan de lanzamiento
├── 04 Producción de contenidos
├── 05 Revisión legal
└── 06 Seguimiento
```

Una conversación por objetivo reduce mezcla de instrucciones y facilita retomar el trabajo.

## 2.9 Continuidad

Al cerrar una conversación larga:

1. resumir decisiones;
2. registrar fuentes utilizadas;
3. listar pendientes;
4. señalar incertidumbres;
5. guardar un documento de continuidad;
6. abrir un chat nuevo con un objetivo específico.

## Práctica guiada

Crear un Project real con:

- instrucciones;
- entre tres y siete fuentes;
- dos conversaciones separadas;
- un documento de continuidad;
- una auditoría de memoria y privacidad;
- una prueba deliberada sobre información ausente.

## Entregable

Project configurado más inventario de fuentes, mapa de conversaciones, reglas de actualización y justificación de qué información no debe vivir en memoria.
